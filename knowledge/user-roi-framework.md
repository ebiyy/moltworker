# ユーザーのタスクROIフレームワーク

## プロフィール

- 収益モデル: フリーランス/受託
- 時給換算: 5,000〜10,000円
- 評価軸: 資本主義的最適化

## 価値評価基準

### 優先度を上げる要因（重要度順）

1. **学習効果** - 将来の単価UPにつながるスキル獲得
2. **納期/デッドライン** - 信用毀損リスク回避
3. **レバレッジ可能** - 記事・実績・テンプレートに転用可能

### 優先度を下げる要因

- アイゼンハワーマトリクス（緊急度×重要度）で判断
- 緊急だが重要でないタスクは外注検討
- どちらでもないタスクは削除候補

## Linear ラベル設計

### 価値ラベル

| ラベル | 意味 |
|--------|------|
| `High-Value` | 直接収益 or 高レバレッジ |
| `Skill-Up` | 学習投資（将来収益） |
| `Reusable` | 転用可能（記事/テンプレ化） |

### 緊急度×重要度ラベル

| ラベル | 意味 | アクション |
|--------|------|-----------|
| `Do-First` | 緊急かつ重要 | 即実行 |
| `Schedule` | 重要だが緊急でない | 計画に入れる |
| `Delegate` | 緊急だが重要でない | 外注検討 |
| `Eliminate` | どちらでもない | やらない |

### 見積もりラベル

| ラベル | 時間 |
|--------|------|
| `Quick` | 30分未満 |
| `Medium` | 1-3時間 |
| `Large` | 1日以上 |

## タスクインタビュー質問

新規タスク追加時に確認すべき項目：

1. **納期は？** → Do-First / Schedule 判定
2. **学習要素ある？** → Skill-Up ラベル判定
3. **転用できそう？** → Reusable ラベル判定
4. **どのくらいかかりそう？** → 見積もりラベル
5. **収益に直結する？** → High-Value ラベル判定
